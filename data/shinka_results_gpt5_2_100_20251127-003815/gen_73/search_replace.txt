<DESCRIPTION>
This edit strengthens the eviction and update logic by introducing a lightweight decayed-recency signal combined with TinyLFU estimates in a lexicographic comparator. Specifically:
- Track last-touch times per key and integrate them with TinyLFU in a lexicographic coldness comparator (lower TinyLFU estimate first; if tied, older last-touch is colder).
- Use this comparator when sampling candidates from SLRU segment tails, with a +1 bias for M2 to protect the main protected segment.
- Pass current time to touch operations and candidate sampling to keep metadata consistent.
- Improve segment size enforcement using while loops and frequency-aware demotions:
  - Demote from W2 to W1 using the cold comparator instead of strict LRU.
  - Demote from M2 to M1 using the cold comparator with M2 bias.
- Ensure on_evict cleans last-touch metadata to avoid stale state growth.
- Make candidate selection in eviction more robust and scan-aware, preferring M1 replacement when new is hotter, otherwise evicting from W1, and touching M2 only if new is significantly hotter.

These changes reduce miss rate by better distinguishing cold items via both frequency and recentness, protecting long-lived hot sets while adapting quickly to phase changes and scans.

        - Maintain W2 within its target (demote W2 LRU back to W1 if needed).
        - Maintain M2 within its target (demote low-freq M2 to M1).
        - Update scan detector.
